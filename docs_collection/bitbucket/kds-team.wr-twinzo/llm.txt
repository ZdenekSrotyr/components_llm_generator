File: README.md
================================================

=============

Twinzo Writer writes data to Twinzo instance.

**Table of contents:**

[TOC]

Functionality notes
===================
This component is currently in development.

Prerequisites
=============

Authentication is done using Authorization endpoint which needs following fields:

- Client
- Login
- Password
- API key

You can find out how to obtain these in [Twinzo Documentation](https://twinzo.eu/slides/slide/api-keys-59).

You can select multiple tables as an input authenticated with the same credentials and API key.


Configuration
=============

## Writer configuration
 - Client (client) - [REQ] Twinzo instance Client Name
 - Login (login) - [REQ] Twinzo instance Login 
 - Password (#password) - [REQ] 
 - API key (#api_key) - [REQ] You can find more info about Twinzo API keys [here](href='https://twinzo.eu/slides/slide/api-keys-59').

## Writer row configuration
 - Writer Type (writer_type) - [REQ] Only sensors are supported at the moment. If you need more endpoints, please submit your request to
[ideas.keboola.com](https://ideas.keboola.com/)


Sample Configuration
=============
```json
{
    "parameters": {
        "client": "FIREHOUSE",
        "login": "Keboola",
        "#password": "SECRET_VALUE",
        "#api_key": "SECRET_VALUE",
        "writer_type": "sensors"
    },
    "action": "run"
}
```

### Sensors data

Endpoint /v3.0/sensors/batch is used to send sensor data in batches.
If the sensor does not exist, the component will create it using v3.0/sensors endpoint.

The input table for sensors has to have the following columns:

 - timestamp: Unix 13 chars format
 - quantity: Quantity of the value, for example "Temperature" or "Humidity" 
 - value: The value of the measurement, can be both in or string.
 - unit: The unit of measurement, for example "°C" or "%"
 - datatype: Datatype of the measurement, please refer to [Twinzo API docs](https://api.twinzo.eu/swagger/ui/index#!/Sensors/Sensors_PostSensorsData) for more information
about supported Datatypes.
 - value_ok: Can be true or false. This can help to identify reaching a certain threshold for a sensor.
 - login: Login name of the sensor.


Development
-----------

If required, change local data folder (the `CUSTOM_FOLDER` placeholder) path to your custom path in
the `docker-compose.yml` file:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
    volumes:
      - ./:/code
      - ./CUSTOM_FOLDER:/data
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Clone this repository, init the workspace and run the component with following command:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
docker-compose build
docker-compose run --rm dev
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Run the test suite and lint check using this command:

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
docker-compose run --rm test
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Integration
===========

For information about deployment and integration with KBC, please refer to the
[deployment section of developers documentation](https://developers.keboola.com/extend/component/deployment/)

File: deploy.sh
================================================
#!/bin/sh
set -e

env

# compatibility with travis and bitbucket
if [ ! -z ${BITBUCKET_TAG} ]
then
	echo "assigning bitbucket tag"
	export TAG="$BITBUCKET_TAG"
elif [ ! -z ${TRAVIS_TAG} ]
then
	echo "assigning travis tag"
	export TAG="$TRAVIS_TAG"
elif [ ! -z ${GITHUB_TAG} ]
then
	echo "assigning github tag"
	export TAG="$GITHUB_TAG"
else
	echo No Tag is set!
	exit 1
fi

echo "Tag is '${TAG}'"

#check if deployment is triggered only in master
if [ ${BITBUCKET_BRANCH} != "master" ]; then
               echo Deploy on tagged commit can be only executed in master!
               exit 1
fi

# Obtain the component repository and log in
echo "Obtain the component repository and log in"
docker pull quay.io/keboola/developer-portal-cli-v2:latest
export REPOSITORY=`docker run --rm  \
    -e KBC_DEVELOPERPORTAL_USERNAME \
    -e KBC_DEVELOPERPORTAL_PASSWORD \
    quay.io/keboola/developer-portal-cli-v2:latest \
    ecr:get-repository ${KBC_DEVELOPERPORTAL_VENDOR} ${KBC_DEVELOPERPORTAL_APP}`

echo "Set credentials"
eval $(docker run --rm \
    -e KBC_DEVELOPERPORTAL_USERNAME \
    -e KBC_DEVELOPERPORTAL_PASSWORD \
    quay.io/keboola/developer-portal-cli-v2:latest \
    ecr:get-login ${KBC_DEVELOPERPORTAL_VENDOR} ${KBC_DEVELOPERPORTAL_APP})

# Push to the repository
echo "Push to the repository"
docker tag ${APP_IMAGE}:latest ${REPOSITORY}:${TAG}
docker tag ${APP_IMAGE}:latest ${REPOSITORY}:latest
docker push ${REPOSITORY}:${TAG}
docker push ${REPOSITORY}:latest

# Update the tag in Keboola Developer Portal -> Deploy to KBC
if echo ${TAG} | grep -c '^v\?[0-9]\+\.[0-9]\+\.[0-9]\+$'
then
    docker run --rm \
        -e KBC_DEVELOPERPORTAL_USERNAME \
        -e KBC_DEVELOPERPORTAL_PASSWORD \
        quay.io/keboola/developer-portal-cli-v2:latest \
        update-app-repository ${KBC_DEVELOPERPORTAL_VENDOR} ${KBC_DEVELOPERPORTAL_APP} ${TAG} ecr ${REPOSITORY}
else
    echo "Skipping deployment to KBC, tag ${TAG} is not allowed."
fi

from keboola.http_client import HttpClient
import json
import csv
import logging

from requests.exceptions import ConnectionError, HTTPError


class TwinzoClientException(Exception):
    pass


def str2bool(string_, default='raise'):
    """
    Convert a string to a bool.

    Parameters
    ----------
    string_ : str
    default : {'raise', False}
        Default behaviour if none of the "true" strings is detected.

    Returns
    -------
    boolean : bool
    """
    true = ['true', 't', '1', 'y', 'yes', 'enabled', 'enable', 'on']
    false = ['false', 'f', '0', 'n', 'no', 'disabled', 'disable', 'off']
    if string_.lower() in true:
        return True
    elif string_.lower() in false or (not default):
        return False
    else:
        raise ValueError('The value \'{}\' cannot be mapped to boolean.'
                         .format(string_))


class TwinzoClient(HttpClient):

    def __init__(self, base_url, api_key, client, login, password):
        self.api_key = api_key
        self.client = client
        self.login = login
        self.password = password
        self.branch_guid = None
        self.client_guid = None
        default_header = {
            "Content-Type": "application/json",
            "Accept": "application/json"
        }
        super().__init__(base_url=base_url, default_http_header=default_header)
        self.auth()

    def auth(self):
        payload = {
            "Client": self.client,
            "Login": self.login,
            "Password": self.password
        }
        try:
            r = self.post("v3.0/Authorization/Authenticate", json=payload)
        except HTTPError as e:
            raise TwinzoClientException("Cannot authenticate, please check credentials and API key.") from e
        self.branch_guid = r.get("Branch")
        self.client_guid = r.get("Client")

    @staticmethod
    def _get_distinct_sensors(table):
        sensors = []
        with open(table.full_path, 'r', encoding='utf-8') as t:
            reader = csv.DictReader(t)
            for row in reader:
                if row["login"] not in sensors:
                    sensors.append(row["login"])
        logging.info(f"Component will process {len(sensors)} sensors.")
        return sensors

    def _get_existing_sensors(self):
        existing_sensors = []
        headers = {
            "Client": self.client_guid,
            "Branch": self.branch_guid,
            "Api-key": self.api_key
        }
        try:
            r = self.get_raw("v3.0/sensors", headers=headers)
        except ConnectionError as e:
            raise TwinzoClientException(f"Connection failed when trying to list existing sensors, "
                                        f"Error: {r.json()}") from e

        parsed_response = r.json()
        for item in parsed_response:
            if item.get("Login") not in existing_sensors:
                existing_sensors.append(item.get("Login"))

        return existing_sensors

    def _create_unexisting_sensors(self, sensors, existing_sensors):
        unexisting_sensors = [sensor for sensor in sensors if sensor not in existing_sensors]
        headers = {
            "Client": self.client_guid,
            "Branch": self.branch_guid,
            "Api-key": self.api_key
        }
        if unexisting_sensors:
            logging.info(f"Component found {len(unexisting_sensors)} sensors that do not exist in Twinzo database. "
                         f"Component will try to create these before sending new data.")
            for sensor in unexisting_sensors:
                data = {
                    "Login": sensor,
                    "Title": sensor
                }
                data = json.dumps(data)
                try:
                    r = self.post("v3.0/sensors", data=data, headers=headers)
                except ConnectionError as e:
                    raise TwinzoClientException(f"Connection failed when trying to list existing sensors, "
                                                f"Error: {r.json()}") from e
            logging.info(f"Component successfully created following sensors: {unexisting_sensors}")

    def post_sensor_data(self, table):
        sensors = self._get_distinct_sensors(table)
        existing_sensors = self._get_existing_sensors()
        self._create_unexisting_sensors(sensors, existing_sensors)

        for sensor in sensors:
            logging.info(f"Processing sensor: {sensor}")
            record_array = []
            d = {
                "Login": str(sensor)
            }
            with open(table.full_path, 'r', encoding='utf-8') as t:
                reader = csv.DictReader(t)
                for row in reader:
                    if row["login"] == sensor:
                        record = {
                            "Timestamp": int(row["timestamp"]),
                            "Quantity": str(row["quantity"]),
                            "Value": row["value"],
                            "Unit": str(row["unit"]),
                            "DataType": str(row["datatype"]),
                            "ValueOK": str2bool(row["value_ok"])
                        }
                        record_array.append(record)
            d["SensorData"] = record_array
            data = json.dumps([d])
            headers = {
                "Client": self.client_guid,
                "Branch": self.branch_guid,
                "Api-key": self.api_key
            }
            r = None
            try:
                r = self.post("v3.0/sensors/batch", data=data, headers=headers)
                print(data)
            except ConnectionError as e:
                raise TwinzoClientException(f"Connection failed when trying to send sensors data, "
                                            f"Error: {r.json()}") from e
            except HTTPError as e:
                raise TwinzoClientException(f"Client failed to upload data {data}, with response: {r}") from e

    def post_devices(self, id, mac, branchid, sectorid, title, note, x, y, appversion, ismoving, fallstatus, battery,
                     devicetypeid, login, position, geofence, geofencerange):

            "X": float(x),
            "Y": float(y),

            "Battery": int(battery),
            "DeviceTypeId": int(devicetypeid),
            "Login": str(login),
            "Position": str2bool(position),
            "Geofence": str2bool(geofence),

        }]

        except ConnectionError as e:
            raise TwinzoClientException("Connection failed when trying to send sensors data") from e

        try:
            r.raise_for_status()
        except HTTPError as e:
            raise TwinzoClientException(f"Bad Request encountered when trying to register device: {r.text}") from e
