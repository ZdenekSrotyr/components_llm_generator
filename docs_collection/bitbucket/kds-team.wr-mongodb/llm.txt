# MongoDB Writer

Keboola component for writing data from Keboola Connection to MongoDB databases.

## Features

- **Multiple operation types**: Insert, Update, and Upsert operations
- **Batch processing**: Configurable batch size for optimal performance
- **Schema validation**: Support for MongoDB schema validation with automatic type conversion
- **Flexible authentication**: Connect via MongoDB URI
- **Data type conversion**: Automatic conversion from CSV strings to MongoDB types (int, double, date, ObjectId, etc.)

Built using [keboola.component](https://pypi.org/project/keboola.component) library for KBC integration.

**Table of contents:**

- [Features](#features)
- [Configuration](#configuration)
- [Operation Types](#operation-types)
- [Development](#development)
- [CI/CD](#cicd)
- [Testing](#testing)

## Configuration

The component accepts the following parameters:

### Authentication
- `#uri` - MongoDB connection URI (encrypted parameter)

### Destination
- `database` - Target MongoDB database name
- `table` - Target collection name
- `operation_type` - Operation type: `insert`, `update`, or `upsert`
- `use_batches` - Enable batch processing (default: `true`)
- `batch_size` - Number of documents per batch (default: `1000`)
- `schema` - Optional MongoDB validation schema for type conversion

### Example Configuration

```json
{
  "authentication": {
    "#uri": "mongodb://username:password@host:27017/database"
  },
  "destination": {
    "database": "myDatabase",
    "table": "myCollection",
    "operation_type": "insert",
    "use_batches": true,
    "batch_size": 1000
  }
}
```

## Operation Types

### Insert
Inserts new documents into the collection. Does not require `_id` field.

### Update
Updates existing documents by `_id`. Requires `_id` column in the input table with valid ObjectId values.

### Upsert
Updates existing documents or inserts new ones. Requires `_id` column:
- If `_id` is provided and exists: updates the document
- If `_id` is empty or doesn't exist: creates a new document with a new ObjectId

## CI/CD

# MongoDB Writer Component - AI Agent Documentation

## Component Overview

**Component ID**: `kds-team.wr-mongodb`
**Component Type**: Writer (Data Destination)
**Platform**: Keboola Connection
**Language**: Python
**Base Library**: keboola.component v1.x

The MongoDB Writer component writes data from Keboola Connection storage tables to MongoDB databases. It supports multiple write operations (insert, update, upsert) with batch processing capabilities and automatic data type conversion based on MongoDB validation schemas.

## Core Functionality

### Supported Operations

1. **Insert**: Add new documents to MongoDB collection
   - Single row insertion
   - Batch insertion for improved performance
   - Configurable batch size (default: 500)

2. **Update**: Modify existing documents by `_id`
   - Requires `_id` column with valid ObjectId values
   - Updates all fields except `_id`
   - Fails if document doesn't exist

3. **Upsert**: Update existing or insert new documents
   - Requires `_id` column
   - Empty `_id` values trigger new document creation with auto-generated ObjectId
   - Valid `_id` values update existing documents or create if not found

### Key Features

- **Batch Processing**: Optional batch writing for performance optimization
- **Schema Validation**: MongoDB validation schema support with automatic type conversion
- **Data Type Conversion**: Automatic conversion of string data to MongoDB types (int, double, bool, date, ObjectId, nested objects)
- **Connection Flexibility**: Supports MongoDB URI connection strings with authentication
- **Error Handling**: Clear distinction between user errors (exit 1) and system errors (exit 2)

## Architecture

### Component Structure

```
src/
├── component.py          # Main component orchestration
├── configuration.py      # Configuration dataclasses
└── client/
    └── client.py        # MongoDB client implementation
```

### Main Classes

#### Component (src/component.py)
- Extends `ComponentBase` from keboola.component
- Orchestrates the entire write workflow
- Handles operation routing (insert/update/upsert)
- Processes CSV input tables row-by-row or in batches

#### MongoDBClient (src/client/client.py)
- Wraps PyMongo MongoClient
- Handles MongoDB write operations
- Manages data type conversions based on validation schema
- Provides custom exceptions for validation and write errors

#### Configuration (src/configuration.py)
- Defines configuration structure using dataclasses
- `Authentication`: MongoDB URI connection string
- `Destination`: Database, table, operation settings, validation schema

## Configuration

### Component Configuration (configSchema.json)

```json
{
  "authentication": {
    "#uri": "mongodb://user:password@host:port/?authMechanism=SCRAM-SHA-256"
  }
}
```

### Row Configuration (configRowSchema.json)

```json
{
  "destination": {
    "database": "my_database",
    "table": "my_collection",
    "use_batches": true,
    "batch_size": 500,
    "operation_type": "insert",
    "schema": "{\"$jsonSchema\": {...}}"
  }
}
```

#### Configuration Parameters

**Authentication Settings**:
- `#uri` (string, required, encrypted): MongoDB connection URI with credentials
  - Format: `mongodb://[username:password@]host[:port]/[?options]`
  - Supports authentication mechanisms (SCRAM-SHA-256, etc.)

**Destination Settings**:
- `database` (string, required): Target MongoDB database name
- `table` (string, required): Target MongoDB collection name (one collection per row)
- `use_batches` (boolean, optional, default: false): Enable batch writing
- `batch_size` (integer, optional, default: 500): Number of rows per batch (only when use_batches=true)
- `operation_type` (enum, required): Write operation type
  - Options: "insert", "update", "upsert"
- `schema` (string, optional): MongoDB validation schema JSON for automatic type conversion

## Input/Output Specifications

### Input Tables

The component processes one input table per row configuration. Input tables are standard Keboola CSV files with headers.

**Requirements by Operation Type**:

- **Insert**: Any columns, no specific requirements
- **Update**: Must contain `_id` column with valid MongoDB ObjectId values
- **Upsert**: Must contain `_id` column (can be empty for new documents)

### Output

No output tables are generated. The component writes directly to MongoDB and logs operation results.

## Data Type Conversion

When a validation schema is provided, the component automatically converts string data from CSV to MongoDB types:

| MongoDB Type | Conversion Logic |
|-------------|------------------|
| string | `str(value)` |
| int/integer | `int(value)` |
| double | `float(value)` |
| bool | `bool(value)` |
| date | `datetime.strptime(value, "%Y-%m-%d")` |
| ObjectId | `bson.ObjectId(value)` |
| object | JSON parse with nested type conversion |

Example validation schema:
```json
{
  "$jsonSchema": {
    "properties": {
      "name": {"bsonType": "string"},
      "age": {"bsonType": "int"},
      "active": {"bsonType": "bool"},
      "metadata": {
        "bsonType": "object",
        "properties": {
          "score": {"bsonType": "double"}
        }
      }
    }
  }
}
```

## Error Handling

### User Exceptions (Exit Code 1)

Raised via `UserException` for configuration or data issues:
- Missing `_id` column for update/upsert operations
- Invalid ObjectId format in `_id` column
- Schema validation failures
- Invalid document structure

### System Exceptions (Exit Code 2)

Unhandled errors indicating system/infrastructure issues:
- MongoDB connection failures
- Network timeouts
- Unexpected PyMongo errors

### Custom Exceptions

- `MongoDBClientException`: General MongoDB write errors
- `MongoDBValidationException`: Schema validation failures

## Code Workflow

### Main Execution Flow

```python
def run(self) -> None:
    # 1. Initialize and validate configuration
    self._init_configuration()

    # 2. Initialize MongoDB client
    self._init_client()

    # 3. Set up data type conversions if schema provided
    self._set_conversion_if_schema_is_defined()

    # 4. Process input tables
    self.process_tables()
```

### Operation-Specific Workflows

**Insert Operation**:
1. Read CSV file with DictReader
2. Convert data types (if schema provided)
3. Insert one document or batch insert multiple documents
4. Log operation details

**Update Operation**:
1. Read CSV file with DictReader
2. Extract `_id` from each row (validate ObjectId)
3. Build query: `{"_id": ObjectId(_id)}`
4. Build update: `{"$set": {...remaining_fields}}`
5. Execute `update_one` operation

**Upsert Operation**:
1. Read CSV file with DictReader
2. Extract `_id` from each row
3. If `_id` is empty: generate new ObjectId
4. If `_id` has value: validate ObjectId format
5. Build update with `$set` operator
6. Execute `update_one` with `upsert=True`

## Common Use Cases

### Use Case 1: Bulk Insert with Type Conversion

**Scenario**: Insert product data with proper types (prices as doubles, quantities as integers)

**Configuration**:
```json
{
  "destination": {
    "database": "ecommerce",
    "table": "products",
    "operation_type": "insert",
    "use_batches": true,
    "batch_size": 1000,
    "schema": "{\"$jsonSchema\": {\"properties\": {\"name\": {\"bsonType\": \"string\"}, \"price\": {\"bsonType\": \"double\"}, \"quantity\": {\"bsonType\": \"int\"}}}}"
```

## Troubleshooting

### Common Issues

**Issue**: "For update operation, the input table has to have a mandatory column '_id'"
- **Cause**: Missing `_id` column in input CSV for update/upsert operation
- **Solution**: Add `_id` column with valid ObjectId values to input table

**Issue**: "For update operation, every record has to have a valid ObjectId '_id'"
- **Cause**: Invalid ObjectId format in `_id` column
- **Solution**: Ensure `_id` values are 24-character hexadecimal strings

**Issue**: "Schema validation failed"
- **Cause**: Data doesn't match MongoDB validation schema
- **Solution**: Check validation schema and ensure data types match, or update schema

**Issue**: Batch insert fails partway through
- **Cause**: One document in batch fails validation
- **Solution**: Review validation schema, check data quality, consider smaller batch sizes

## Performance Considerations

### Batch Writing

- Enable `use_batches: true` for large datasets (>1000 rows)
- Optimal batch size: 500-1000 rows (balance memory vs. network overhead)
- Trade-off: Larger batches = faster but more memory, entire batch fails if one document invalid

### Data Type Conversion

- Schema-based conversion adds processing overhead (~10-20% slower)
- Only use validation schema when type conversion is necessary
- Pre-process data in Keboola transformation if complex conversions needed

### Connection Pooling

- MongoClient maintains internal connection pool
- Single client instance reused across all operations
- No need for manual connection management

## Security Considerations

- MongoDB URI stored encrypted in Keboola (`#uri` parameter)
- Supports authentication mechanisms (SCRAM-SHA-256 recommended)
- Use read-write user with database-level permissions (not admin)
- Enable MongoDB access control in production environments
- Consider network security (VPN, IP whitelisting) for cloud deployments

import csv
import logging

import bson.errors
from keboola.component.base import ComponentBase
from keboola.component.exceptions import UserException

from client import MongoDBClient, MongoDBClientException, MongoDBValidationException
from configuration import Configuration

from bson import ObjectId


class Component(ComponentBase):
    def __init__(self):
        self._configuration: Configuration
        self.client: MongoDBClient
        self.table_definitions = []
        super().__init__()

    def run(self) -> None:
        self._init_configuration()
        self._init_client()
        self._set_conversion_if_schema_is_defined()
        self.process_tables()

    def _init_configuration(self) -> None:
        self.validate_configuration_parameters(Configuration.get_dataclass_required_parameters())
        self._configuration: Configuration = Configuration.load_from_dict(self.configuration.parameters)

    def _init_client(self) -> None:
        uri = self._configuration.authentication.pswd_uri
        database = self._configuration.destination.database
        self.client = MongoDBClient(uri=uri, database=database)

    def process_tables(self) -> None:
        operation_type = self._configuration.destination.operation_type
        table_name = self._configuration.destination.table

        logging.info(f"Starting operation type: {operation_type}")

        tables = self.get_input_tables_definitions()
        table = tables[0]

        if operation_type == "insert":
            try:
                if self._configuration.destination.use_batches:
                    self.insert_rows_in_batches(table, table_name)
                else:
                    self.insert_rows(table, table_name)
            except (MongoDBClientException, MongoDBValidationException) as e:
                raise UserException(e) from e
        elif operation_type == "update":
            try:
                self.update_rows(table, table_name)
            except (MongoDBClientException, MongoDBValidationException) as e:
                raise UserException(e) from e
        elif operation_type == "upsert":
            try:
                self.upsert_rows(table, table_name)
            except (MongoDBClientException, MongoDBValidationException) as e:
                raise UserException(e) from e

    def update_rows(self, table, table_name):
        with open(table.full_path, 'r') as file:
            reader = csv.DictReader(file)
            for row in reader:
                row_copy = row.copy()

                try:
                    _id = row_copy.pop("_id")
                except KeyError:
                    raise UserException("For update operation, the input table has to have a mandatory column '_id'.")

                try:
                    query = {"_id": ObjectId(_id)}
                except bson.errors.InvalidId:
                    raise UserException("For update operation, every record has to have a valid ObjectId '_id'.")

                update = {
                    "$set": {**row_copy}
                }  # Define the modifications to apply to the selected document(s)

                logging.debug(f"Updating record using query: {query}")
                self.client.update_row(query, update, table_name)

    def upsert_rows(self, table, table_name):
        with open(table.full_path, 'r') as file:
            reader = csv.DictReader(file)
            for row in reader:
                row_copy = row.copy()

                try:
                    _id = row_copy.pop("_id")
                except KeyError:
                    raise UserException("For upsert operation, the input table has to have a mandatory column '_id'.")

                update = {
                    "$set": {**row_copy}
                }

                if _id != "":
                    try:
                        query = {"_id": ObjectId(_id)}
                        logging.debug(f"Upserting update {update} using query: {query}")
                    except bson.errors.InvalidId:
                        raise UserException(f"_id {_id} in row {row} is not valid.")

                    self.client.update_row(query, update, table_name, upsert=True)
                else:
                    query = {"_id": ObjectId()}
                    logging.debug(f"Upserting update {update} using query: {query}")
                    self.client.update_row(query, update, table_name, upsert=True)

    def insert_rows(self, table, table_name) -> None:
        with open(table.full_path, 'r') as file:
            reader = csv.DictReader(file)
            for row in reader:
                self.client.write_row(row, table_name)

    def insert_rows_in_batches(self, table, table_name) -> None:
        batch_size = self._configuration.destination.batch_size
        rows = []
        with open(table.full_path, 'r') as file:
            reader = csv.DictReader(file)
            for row in reader:
                rows.append(row)
                if len(rows) >= batch_size:
                    self.client.write_rows(rows, table_name)
                    logging.debug(f"Batch sent: {rows}")
                    rows = []
            if rows:
                self.client.write_rows(rows, table_name)
                logging.debug(f"Final batch sent: {rows}")

    def _set_conversion_if_schema_is_defined(self):
        if schema := self._configuration.destination.schema:
            self.client.set_conversion(schema)


"""
        Main entrypoint
"""
if __name__ == "__main__":
    try:
        comp = Component()
        comp.execute_action()
    except UserException as exc:
        logging.exception(exc)
        exit(1)
    except Exception as exc:
        logging.exception(exc)
        exit(2)

import dataclasses
import json
from dataclasses import dataclass
from typing import List

import dataconf


class ConfigurationBase:
    @staticmethod
    def _convert_private_value(value: str):
        return value.replace('"#', '"pswd_')

    @staticmethod
    def _convert_private_value_inv(value: str):
        if value and value.startswith("pswd_"):
            return value.replace("pswd_", "#", 1)
        else:
            return value

    @classmethod
    def load_from_dict(cls, configuration: dict):
        """
        Initialize the configuration dataclass object from dictionary.
        Args:
            configuration: Dictionary loaded from json configuration.

        Returns:

        """
        json_conf = json.dumps(configuration)
        json_conf = ConfigurationBase._convert_private_value(json_conf)
        return dataconf.loads(json_conf, cls, ignore_unexpected=True)

    @classmethod
    def get_dataclass_required_parameters(cls) -> List[str]:
        """
        Return list of required parameters based on the dataclass definition (no default value)
        Returns: List[str]

        """
        return [cls._convert_private_value_inv(f.name)
                for f in dataclasses.fields(cls)
                if f.default == dataclasses.MISSING
                and f.default_factory == dataclasses.MISSING]


@dataclass
class Authentication(ConfigurationBase):
    pswd_uri: str


@dataclass
class Destination(ConfigurationBase):
    database: str
    table: str
    use_batches: bool = True
    batch_size: int = 1000
    schema: str = None
    operation_type: str = None


@dataclass
class Configuration(ConfigurationBase):
    authentication: Authentication
    destination: Destination

from bson.errors import InvalidDocument
from pymongo import MongoClient
from pymongo.errors import BulkWriteError, WriteError
from typing import List, Dict, Union
import json
import logging
from datetime import datetime
import bson


class MongoDBClientException(Exception):
    pass


class MongoDBValidationException(Exception):
    pass


class MongoDBClient:
    def __init__(self, host=None, port=None, database=None, uri=None):
        if uri:
            self.client = MongoClient(uri)
        else:
            self.client = MongoClient(host, port)
        self.database = self.client[database]
        self.conversions = None

    def authenticate(self, username, password, mechanism: str = 'SCRAM-SHA-256') -> None:
        self.database.authenticate(username, password, mechanism=mechanism)

    def write_row(self, row: Union[dict, str], table_name: str) -> None:
        converted_row = self.convert_datatypes(row)
        try:
            self.database[table_name].insert_one(converted_row)
            logging.debug(
                f"Row sent: {converted_row}, "
                f"Types: {', '.join(f'{converted_row}: {type(value)}' for key, value in converted_row.items())}")
        except InvalidDocument as e:
            raise MongoDBClientException(f"Cannot write {row} to database.") from e
        except WriteError as e:
            if e.details.get("errmsg"):
                if e.details.get("errInfo"):
                    raise MongoDBValidationException(f"Schema validation failed: {e.details.get('errInfo')}")
            raise MongoDBClientException("An error occurred while writing to the database.") from e

    def write_rows(self, rows: Union[List[Dict[str, any]], List[str]], table_name: str) -> None:
        try:
            converted_rows = []
            for row in rows:
                converted_row = self.convert_datatypes(row)
                converted_rows.append(converted_row)
            self.database[table_name].insert_many(converted_rows)
        except InvalidDocument as e:
            raise MongoDBClientException(f"Cannot write {rows} to the database.") from e
        except BulkWriteError as e:
            if e.details.get('writeErrors'):
                error = e.details['writeErrors'][0]
                if error.get("errmsg") == "Document failed validation":
                    raise MongoDBValidationException(f"Schema validation failed: {error}")
            raise MongoDBClientException("An error occurred while writing to the database.") from e

    def update_row(self, query: dict, update: dict, table_name: str, upsert: bool = False) -> None:
        update["$set"] = self.convert_datatypes(update["$set"])
        try:
            self.database[table_name].update_one(query, update, upsert=upsert)
        except WriteError as e:
            if e.details.get("errmsg"):
                if e.details.get("errInfo"):
                    raise MongoDBValidationException(f"Schema validation failed: {e.details.get('errInfo')}")
            raise MongoDBClientException("An error occurred while updating the row.") from e

    def set_conversion(self, schema_string):
        conversion_dict = {}

        data = json.loads(schema_string)

        json_schema = data.get("$jsonSchema", {})
        properties = json_schema.get("properties", {})

        if properties:
            for column, column_props in properties.items():
                bson_type = column_props.get("bsonType")
                if bson_type:
                    if bson_type == "object":
                        conversion_dict[column] = {"type": bson_type}
                        sub_properties = column_props.get("properties")
                        if sub_properties:
                            for sub_column, sub_column_props in sub_properties.items():
                                sub_bson_type = sub_column_props.get("bsonType")
                                if sub_bson_type:
                                    conversion_dict[column][sub_column] = {"type": sub_bson_type}
                    else:
                        conversion_dict[column] = {"type": bson_type}

        if conversion_dict:
            self.conversions = conversion_dict

        logging.info(f"Conversions set using validation schema: {conversion_dict}")

    def _convert_datatypes(self, conversion_info: dict, column, value):
        converted_field = {}

        logging.debug(f"Processing: {column, value} using {conversion_info}")

        if "type" in conversion_info:
            target_type = conversion_info["type"]
            if target_type == 'string':
                converted_field[column] = str(value)
            elif target_type in ['int', 'integer']:
                converted_field[column] = int(value)
            elif target_type == 'double':
                converted_field[column] = float(value)
            elif target_type == 'bool':
                converted_field[column] = bool(value)
            elif target_type == 'date':
                converted_field[column] = datetime.strptime(value,
                                                            "%Y-%m-%d")  # Adjust the date format as needed
            elif target_type == 'ObjectId':
                converted_field[column] = bson.ObjectId(value)
            elif target_type == 'object':
                # TODO: handle nested datatype mapping
                new_val = value.replace("'", "\"")
                d = json.loads(new_val)

                converted_field[column] = {}

                for sub_column, sub_value in d.items():
                    converted_value = self._convert_datatypes(
                        conversion_info[sub_column], sub_column, sub_value)
                    converted_field[column][sub_column] = converted_value[sub_column]

        else:
            # If "type" is not specified, fallback to the original value
            return value
        return converted_field

    def convert_datatypes(self, row: dict):
        converted_data = {}

        if self.conversions:
            for column, value in row.items():
                if column in self.conversions:
                    conversion_info = self.conversions[column]
                    converted_field = self._convert_datatypes(conversion_info, column, value)
                    converted_data[column] = converted_field[column]
                else:
                    converted_data[column] = value
            return converted_data
        return row

name: Keboola Component Build & Deploy Pipeline
on:
  push:  # skip the workflow on the main branch without tags
    branches-ignore:
      - master
    tags:
      - "*"

concurrency: ci-${{ github.ref }}  # to avoid tag collisions in the ECR
env:
  # repository variables:
  KBC_DEVELOPERPORTAL_APP: "keboola.wr-mongo"
  KBC_DEVELOPERPORTAL_VENDOR: "keboola"
  DOCKERHUB_USER: ${{ secrets.DOCKERHUB_USER }}
  KBC_DEVELOPERPORTAL_USERNAME: ${{ vars.KBC_DEVELOPERPORTAL_USERNAME }}

  # repository secrets:
  DOCKERHUB_TOKEN: ${{ secrets.DOCKERHUB_TOKEN }}  # recommended for pushing to ECR
  KBC_DEVELOPERPORTAL_PASSWORD: ${{ secrets.KBC_DEVELOPERPORTAL_PASSWORD }}

  # (optional) test KBC project: https://connection.keboola.com/admin/projects/0000
  KBC_TEST_PROJECT_CONFIGS: ""  # space separated list of config ids
  KBC_STORAGE_TOKEN: ${{ secrets.KBC_STORAGE_TOKEN }}  # required for running KBC tests

jobs:
  push_event_info:
    name: Push Event Info
    runs-on: ubuntu-latest
    outputs:
      app_image_tag: ${{ steps.tag.outputs.app_image_tag }}
      is_semantic_tag: ${{ steps.tag.outputs.is_semantic_tag }}
      is_default_branch: ${{ steps.default_branch.outputs.is_default_branch }}
      is_deploy_ready: ${{ steps.deploy_ready.outputs.is_deploy_ready }}
    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4

      - name: Fetch all branches from remote repository
        run: git fetch --prune --unshallow --tags -f

      - name: Get current branch name
        id: current_branch
        run: |
          if [[ ${{ github.ref }} != "refs/tags/"* ]]; then
            branch_name=${{ github.ref_name }}
            echo "branch_name=$branch_name" | tee -a $GITHUB_OUTPUT
          else
            raw=$(git branch -r --contains ${{ github.ref }})
            branch="$(echo ${raw/*origin\//} | tr -d '\n')"
            echo "branch_name=$branch" | tee -a $GITHUB_OUTPUT
          fi

      - name: Is current branch the default branch
        id: default_branch
        run: |
          echo "default_branch='${{ github.event.repository.default_branch }}'"
          if [ "${{ github.event.repository.default_branch }}" = "${{ steps.current_branch.outputs.branch_name }}" ]; then
             echo "is_default_branch=true" | tee -a $GITHUB_OUTPUT
          else
             echo "is_default_branch=false" | tee -a $GITHUB_OUTPUT
          fi

      - name: Set image tag
        id: tag
        run: |
          TAG="${GITHUB_REF##*/}"
          IS_SEMANTIC_TAG=$(echo "$TAG" | grep -q '^v\?[0-9]\+\.[0-9]\+\.[0-9]\+$' && echo true || echo false)
          echo "is_semantic_tag=$IS_SEMANTIC_TAG" | tee -a $GITHUB_OUTPUT
          echo "app_image_tag=$TAG" | tee -a $GITHUB_OUTPUT

      - name: Deploy-Ready check
        id: deploy_ready
        run: |
          if [[ "${{ steps.default_branch.outputs.is_default_branch }}" == "true" \
            && "${{ github.ref }}" == refs/tags/* \
            && "${{ steps.tag.outputs.is_semantic_tag }}" == "true" ]]; then
              echo "is_deploy_ready=true" | tee -a $GITHUB_OUTPUT
          else
              echo "is_deploy_ready=false" | tee -a $GITHUB_OUTPUT
          fi

  build:
    name: Docker Image Build
    runs-on: ubuntu-latest
    needs:
      - push_event_info
    env:
      DOCKERHUB_TOKEN: ${{ secrets.DOCKERHUB_TOKEN }}
    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          file: ./Dockerfile
          tags: ${{ env.KBC_DEVELOPERPORTAL_APP }}:latest
          outputs: type=docker,dest=/tmp/${{ env.KBC_DEVELOPERPORTAL_APP }}.tar

      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: ${{ env.KBC_DEVELOPERPORTAL_APP }}
          path: /tmp/${{ env.KBC_DEVELOPERPORTAL_APP }}.tar

  tests:
    name: Run Tests
    runs-on: ubuntu-latest
    needs:
      - push_event_info
      - build
    steps:
      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Download artifact
        uses: actions/download-artifact@v4
        with:
          name: ${{ env.KBC_DEVELOPERPORTAL_APP }}
          path: /tmp

      - name: Load Image & Run Tests
        run: |
          docker load --input /tmp/${{ env.KBC_DEVELOPERPORTAL_APP }}.tar
          docker image ls -a
          docker run ${{ env.KBC_DEVELOPERPORTAL_APP }}:latest ruff check /code/src/
          docker run ${{ env.KBC_DEVELOPERPORTAL_APP }}:latest ruff format --check /code/src/
          echo "Running unit-tests..."
          docker run ${{ env.KBC_DEVELOPERPORTAL_APP }}:latest python -m unittest discover

  tests-kbc:
    name: Run KBC Tests
    needs:
      - push_event_info
      - build
    runs-on: ubuntu-latest
    steps:
      - name: Set up environment variables
        run: |
          echo "KBC_TEST_PROJECT_CONFIGS=${KBC_TEST_PROJECT_CONFIGS}" >> $GITHUB_ENV
          echo "KBC_STORAGE_TOKEN=${{ secrets.KBC_STORAGE_TOKEN }}" >> $GITHUB_ENV

      - name: Run KBC test jobs
        if: env.KBC_TEST_PROJECT_CONFIGS != '' && env.KBC_STORAGE_TOKEN != ''
        uses: keboola/action-run-configs-parallel@master
        with:
          token: ${{ secrets.KBC_STORAGE_TOKEN }}
          componentId: ${{ env.KBC_DEVELOPERPORTAL_APP }}
          tag: ${{ needs.push_event_info.outputs.app_image_tag }}
          configs: ${{ env.KBC_TEST_PROJECT_CONFIGS }}

  push:
    name: Docker Image Push
    runs-on: ubuntu-latest
    needs:
      - push_event_info
      - tests
      - tests-kbc
    env:
      DOCKERHUB_TOKEN: ${{ secrets.DOCKERHUB_TOKEN }}
    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4

      - name: Download artifact
        uses: actions/download-artifact@v4
        with:
          name: ${{ env.KBC_DEVELOPERPORTAL_APP }}
          path: /tmp

      - name: Load Image
        run: |
          docker load --input /tmp/${{ env.KBC_DEVELOPERPORTAL_APP }}.tar
          docker image ls -a

      - name: Docker login
        if: env.DOCKERHUB_TOKEN
        run: docker login --username "${{ env.DOCKERHUB_USER }}" --password "${{ env.DOCKERHUB_TOKEN }}"

      - name: Push image to ECR
        uses: keboola/action-push-to-ecr@master
        with:
          vendor: ${{ env.KBC_DEVELOPERPORTAL_VENDOR }}
          app_id: ${{ env.KBC_DEVELOPERPORTAL_APP }}
          username: ${{ env.KBC_DEVELOPERPORTAL_USERNAME }}
          password: ${{ secrets.KBC_DEVELOPERPORTAL_PASSWORD }}
          tag: ${{ needs.push_event_info.outputs.app_image_tag }}
          push_latest: ${{ needs.push_event_info.outputs.is_deploy_ready }}
          source_image: ${{ env.KBC_DEVELOPERPORTAL_APP }}

  deploy:
    name: Deploy to KBC
    env:
      KBC_DEVELOPERPORTAL_PASSWORD: ${{ secrets.KBC_DEVELOPERPORTAL_PASSWORD }}
    needs:
      - push_event_info
      - build
      - push
    if: needs.push_event_info.outputs.is_deploy_ready == 'true'
    runs-on: ubuntu-latest
    steps:
      - name: Set Developer Portal Tag
        uses: keboola/action-set-tag-developer-portal@master
        with:
          vendor: ${{ env.KBC_DEVELOPERPORTAL_VENDOR }}
          app_id: ${{ env.KBC_DEVELOPERPORTAL_APP }}
          username: ${{ env.KBC_DEVELOPERPORTAL_USERNAME }}
          password: ${{ secrets.KBC_DEVELOPERPORTAL_PASSWORD }}
          tag: ${{ needs.push_event_info.outputs.app_image_tag }}

  update_developer_portal_properties:
    name: Developer Portal Properties Update
    env:
      KBC_DEVELOPERPORTAL_PASSWORD: ${{ secrets.KBC_DEVELOPERPORTAL_PASSWORD }}
    needs:
      - push_event_info
      - build
      - push
    runs-on: ubuntu-latest
    if: needs.push_event_info.outputs.is_deploy_ready == 'true'
    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4

      - name: Update developer portal properties
        run: |
          chmod +x scripts/update_dev_portal_properties.sh
          scripts/update_dev_portal_properties.sh
